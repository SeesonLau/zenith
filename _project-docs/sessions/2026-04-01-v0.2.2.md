# Session Log — 2026-04-01 — v0.2.2

## What Was Done

### 1. Leisure Analytics — Expandable By Type
- Monthly and Overall "By Type" rows are now tappable
- Tap a type → expands to show all titles for that type, sorted by session count (most used first), with `Nx · Xh Xm` summary
- Tap a title → expands to show every session: date, time, duration, notes (if any), sorted newest first
- Both levels are independently toggleable; state is scoped per tab (monthly vs overall) so they don't interfere
- Added `buildTitlesByType` helper, `monthTitlesByType` and `overallTitlesByType` memos

### 2. Leisure Analytics Heatmap UX
- Wrapped Session Count heatmap in a horizontal `ScrollView` — all 24 columns now reachable without clipping
- Day/Night divider: dark teal (`#0f766e`) 1px vertical line visible in every cell row

### 3. Leisure Session Edit/Delete — Finance Pattern
- `leisure/[id].tsx` rewritten: view mode (default) + edit mode (toggled by Edit button in header)
- Edit fields: Content Type (LeisureTypePicker), Title, Notes, Date (DateTimePicker), Time (DateTimePicker), Duration (hours + minutes)
- Date & Time placed at the very top of the edit form
- Cancel in edit mode resets all fields to original values
- Delete button remains at bottom of view mode detail screen
- `updateLeisureLog` added to `leisureActions.ts` — updates type/title/notes/startedAt/duration, recomputes endedAt, marks isSynced=false
- Removed standalone `edit.tsx` route (was created then deleted in same session)

### 4. LeisureTypePicker Default State
- All groups (I, II, III) now start collapsed — was Group I open by default
- `complete.tsx`: type state now starts as `null` instead of `'Manga'`; Save is blocked with an alert if no type selected
- `[id].tsx` edit mode: passes `editType` (always has a value loaded from the log)

### 5. Infinite Scroll / Lazy Loading
- **Leisure tab**: removed `useCompletedLeisureLogs(20)` cap; now uses `useAllLeisureLogs()` filtered to completed, sorted newest first; paginated in groups of 15 date-groups
- **Finance tab**: `groupedLogs` (all-time transactions) now paginated — 15 groups initially, loads 15 more on scroll
- **Habit History**: grouped session list paginated — 15 groups initially, resets to 15 when period (Week/Month/All) changes
- Pattern: `onScroll` on outer `ScrollView` with `scrollEventThrottle={200}`, triggers load when within 400px of bottom
